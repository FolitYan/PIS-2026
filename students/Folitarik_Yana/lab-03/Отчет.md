## Цель работы

Научиться применять тактические паттерны DDD (Entities, Value Objects, Aggregates, Domain Events) для реализации доменного слоя системы, обеспечивая высокую степень изоляции бизнес-логики и защиту инвариантов.

**Вариант №1 — Post Service (Система публикаций)**  
- Питч: Платформа для публикации коротких заметок и взаимодействия пользователей через лайки.  
- Ядро домена: Агрегат `Post`, объекты-значения `PostContent`, `AuthorId`, перечисление `PostStatus`.

---

## Ход выполнения работы

### 1. Value Objects (Объекты-значения)

Все Value Objects реализованы с использованием **Java Records**, что гарантирует их неизменяемость (immutability) и автоматическую реализацию методов сравнения.

- **`AuthorId`** — идентификатор автора публикации.  
  - Валидация: строка не может быть пустой или состоять только из пробелов.  
  - Иммутабельность: ✅  
  - Файл: `domain/valueobject/AuthorId.java`

- **`PostContent`** — текстовое содержимое публикации.  
  - Валидация: длина текста должна быть от 1 до 280 символов.  
  - Иммутабельность: ✅  
  - Файл: `domain/valueobject/PostContent.java`

**Пример кода (`PostContent`):**
```java
public record PostContent(String value) {
    public PostContent {
        if (value == null || value.isBlank() || value.length() > 280) {
            throw new DomainException("Контент должен быть от 1 до 280 символов");
        }
    }
}
```

### 2. Entities (Сущности)

Созданная Entity:

- **`Post`** — корневая сущность (Aggregate Root).  
  - ID поле: `UUID id`.  
  - Бизнес-правила: инкапсуляция состояния, управление списком доменных событий.  
  - Файл: `domain/model/Post.java`

**Пример кода (фрагмент инвариантов):**
```java
public void addLike() {
    // Инвариант: нельзя лайкать пост, если он не опубликован
    if (this.status != PostStatus.PUBLISHED) {
        throw new DomainException("Нельзя поставить лайк неопубликованному посту");
    }
    this.likesCount++;
}
```

### 3. Aggregate Root (Корневой агрегат)

**Aggregate Root:** `Post` (Публикация)

**Границы агрегата:**
- Корень: `Post`
- Value Objects: `AuthorId`, `PostContent`, `PostStatus`
- События: `PostCreatedEvent`

**Инварианты агрегата:**

| № | Инвариант | Как проверяется |
|---|-----------|------------------|
| 1 | Текст поста не может быть пуст или > 280 симв. | В конструкторе `PostContent` |
| 2 | Пост создается только в статусе `DRAFT` | В конструкторе `Post` |
| 3 | Лайк возможен только для статуса `PUBLISHED` | В методе `addLike()` |
| 4 | Публикация возможна только из статуса `DRAFT` | В методе `publish()` |

### 4. Domain Events (Доменные события)

Созданное событие:

- **`PostCreatedEvent`** — генерируется при успешном создании нового агрегата `Post`.  
  - Данные: `postId`, `authorId`, `occurredOn`.  
  - Файл: `domain/event/PostCreatedEvent.java`

**Пример кода события:**
```java
public record PostCreatedEvent(UUID postId, String authorId, Instant occurredOn) 
    implements DomainEvent {
    public PostCreatedEvent(UUID postId, String authorId) {
        this(postId, authorId, Instant.now());
    }
}
```

## Контрольные вопросы

**1. В чём отличие Value Object от Entity?**  
Entity (сущность) имеет уникальный ID, по которому она идентифицируется на протяжении всего жизненного цикла (даже если остальные поля изменятся). Value Object (объект-значение) не имеет ID; он определяется совокупностью своих свойств. Если два VO имеют одинаковые значения полей — это один и тот же объект.

**2. Почему Aggregate Root должен инкапсулировать доступ к внутренним сущностям?**  
Чтобы гарантировать целостность данных и соблюдение бизнес-правил (инвариантов). Все изменения внутри агрегата должны проходить через корень, который проверяет возможность выполнения операции.

**3. Какая роль Domain Events?**  
Доменные события уведомляют другие части системы (или другие агрегаты) о значимых изменениях в состоянии домена. Пример: при создании поста может потребоваться обновить ленту подписчиков или отправить уведомление.

**4. Как вы проверяете инварианты в вашем агрегате?**  
Инварианты проверяются внутри методов агрегата и конструкторов Value Objects. Например, метод `addLike()` выбрасывает `DomainException`, если статус поста не соответствует `PUBLISHED`.

**5. Почему Value Objects делаются иммутабельными?**  
Это исключает побочные эффекты. Поскольку VO идентифицируются по значению, их изменение привело бы к концептуальной замене объекта. Иммутабельность делает код более безопасным и предсказуемым.
