## Цель работы

Создать комплексную стратегию тестирования для **Post Service**, включающую юнит-тестирование домена и бизнес-логики, интеграционное тестирование базы данных и сквозное (E2E) тестирование API.

**Вариант №20 — Post Service (Система публикаций)**  
- Ядро домена: Агрегат `Post`, объекты-значения `PostContent`, `AuthorId`.  
- Технологический стек: Java 17, Spring Boot 3.2.2, JUnit 5, Mockito, Testcontainers (PostgreSQL).

---

## Ход выполнения работы

### 1. Юнит-тесты (Domain Layer)

**Файл:** `PostTest.java`  
**Покрытие домена:** 100% бизнес-правил.

**Реализованные сценарии:**
- Проверка инварианта: текст поста не может быть пустым или длиннее 280 символов.
- Проверка состояний: пост создается в статусе `DRAFT`.
- Проверка правил: нельзя поставить лайк посту, который не опубликован (`PUBLISHED`).
- Регистрация доменного события `PostCreatedEvent` при создании агрегата.

**Пример кода теста:**
```java
@Test
void shouldThrowExceptionWhenContentIsTooLong() {
    String longText = "a".repeat(281);
    assertThrows(DomainException.class, () -> new PostContent(longText));
}
```

---

### 2. Юнит-тесты (Application Layer)

**Файл:** `CreatePostHandlerTest.java`  
**Инструментарий:** Mockito (Mocking).

В прикладном слое протестирована оркестрация команды `CreatePostCommand`. С помощью Mockito были изолированы (замоканы) исходящие порты `PostRepository` и `EventPublisher`.

**Результат:** Тест подтверждает, что при вызове хэндлера агрегат создается, сохраняется в репозиторий, а доменные события успешно публикуются во внешний мир.

---

### 3. Интеграционные тесты (Infrastructure Layer)

**Файл:** `PostRepositoryAdapterIT.java`  
**Технология:** Testcontainers (PostgreSQL 15).

Вместо использования In-Memory БД (H2), применены реальные контейнеры. Это гарантирует, что SQL-запросы, маппинг `PostJpaEntity` и работа рефлексии в адаптере корректно работают с PostgreSQL.

**Проверенные методы:**
- `save()` — корректное сохранение агрегата и связанных полей.
- `findById()` — восстановление агрегата из БД с сохранением его состояния (likes, status) через рефлексию.

---

### 4. E2E-тесты (Сквозной сценарий)

**Файл:** `PostApiE2ETest.java`  
**Инструментарий:** `TestRestTemplate`, `@SpringBootTest`.

**Сценарий полного цикла:**
1. `POST /api/posts` — клиент отправляет JSON с ID автора и текстом. Система возвращает UUID созданного поста (201 Created).
2. `GET /api/posts/{id}` — клиент запрашивает данные по полученному ID. Система возвращает DTO `PostResponse`.

**Результат:** Подтверждена работоспособность всей цепочки:  
`Controller → Handler → Domain → Repository → Database`.
